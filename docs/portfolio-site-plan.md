# Portfolio Website Plan

## Direction
- Build a flat static portfolio site.
- Root pages: `index.html`, `contact.html`, `resume.html`.
- Projects live under `projects/` with one subfolder per project.

## Positioning
- Primary identity: iOS software engineer with deep experience since 2001.
- Secondary strengths: Python automation and ML experimentation.
- Current transition: AI integration engineering focused on RAG and agent systems.

## Content Strategy
- Keep project claims credible and prototype-oriented where applicable.
- Include historical credibility: Windows shareware business, international sales, and magazine pickup.
- Resume page includes HTML highlights and downloadable PDF.

## Information Architecture
- Home: narrative, focus areas, timeline, featured projects.
- Projects: index + detailed case study pages.
- Resume: highlights + embedded/downloadable PDF.
- Contact: collaboration focus and channels.
